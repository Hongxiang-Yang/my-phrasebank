# Academic Phrasebank MVP Specification

## 1. Product Name

Working name: **My Phrasebank**

A personal academic writing phrasebank for a Chinese native speaker writing in English. The tool helps the user manually collect English phrases, define their meaning, group them by Chinese categories, and later browse them while writing.

The app is not designed to automatically translate Chinese intentions into English. The user manually enters English phrases and English explanations. AI is used only to generate example sentences, usage scenarios, and optional style notes based on the user’s saved phrase.

---

## 2. Core Problem

The user often feels that their academic English sounds too Chinese, too stiff, or too AI-like. They want a personal database of useful English phrases, linking expressions, discourse moves, and natural academic wording.

The user wants to build their own phrasebank over time and browse phrases by Chinese meaning categories while writing.

---

## 3. MVP Goals

The MVP should support four main tasks:

1. Add and edit phrase entries manually.
2. Let AI generate example sentences and suitable usage scenarios from a saved phrase.
3. Browse phrases by Chinese categories.
4. Work well on both desktop web and mobile screens.

---

## 4. Core Pages / Windows

### Page 1: Phrase Entry / Word Card Editor

This is the main “word window” where the user adds or edits one phrase.

Each phrase card should contain:

- English phrase
- English definition written by the user
- Chinese category
- Optional Chinese meaning note
- Usage type
- Academic tone
- Example sentences generated by AI
- Usage scenarios generated by AI
- User notes
- Tags
- Favourite status
- Created date
- Updated date

Suggested fields:

```json
{
  "id": "uuid",
  "phrase": "This does not necessarily mean that...",
  "definition": "Used to soften or qualify an interpretation before offering a more precise explanation.",
  "chineseCategory": "限定 / 避免绝对化",
  "chineseNote": "不是说……；不一定意味着……",
  "usageType": "discussion",
  "tone": "cautious academic",
  "tags": ["discussion", "qualification", "less-ai"],
  "examples": [
    "This does not necessarily mean that users rejected the avatar, but rather that the avatar made certain aspects of identity less available.",
    "This does not necessarily mean that prior findings are incorrect. Instead, it suggests that the effect may depend on the social context."
  ],
  "scenarios": [
    "Use when explaining a finding carefully.",
    "Use when avoiding an over-strong claim.",
    "Useful in Discussion sections after presenting a surprising result."
  ],
  "notes": "Good for CHI-style cautious interpretation.",
  "isFavourite": true,
  "createdAt": "date",
  "updatedAt": "date"
}
```

The user should be able to:

- Create a new phrase.
- Edit an existing phrase.
- Delete a phrase.
- Save a phrase.
- Mark a phrase as favourite.
- Click “Generate examples with AI”.
- Regenerate examples.
- Copy phrase.
- Copy example sentence.

## 5. AI Feature

AI should only work after the user has manually entered:

- English phrase
- English definition

Optional extra inputs:

- Chinese category
- Chinese meaning note
- Usage type
- Tone

AI output should include:

1. 3 academic example sentences.
2. 3 possible usage scenarios.
3. 1 short warning about misuse, if relevant.
4. Optional style note, such as “good for cautious discussion” or “too formal for email”.

Example AI prompt logic:

```text
You are helping the user build a personal academic phrasebank.

Given the following English phrase and user-written definition, generate:
1. Three natural academic example sentences.
2. Three suitable usage scenarios.
3. One misuse warning if needed.
4. One short style note.

Do not translate from Chinese.
Do not generate unrelated phrases.
Do not overuse generic AI-style academic wording.
Keep examples suitable for HCI / Social VR / qualitative research when possible.

Phrase: {phrase}
Definition: {definition}
Chinese category: {chineseCategory}
Chinese note: {chineseNote}
Usage type: {usageType}
Tone: {tone}
```

AI output should be editable by the user before saving.

The app should have a simple navigation structure:

```text
Home
├── Add Phrase
├── Browse by Category
├── Favourites
└── Settings
```

### Home

Show:

- Total number of phrases
- Recently added phrases
- Favourite phrases
- Quick button: Add Phrase
- Quick button: Browse Categories

### Add Phrase

Open the Phrase Entry / Word Card Editor.

### Browse by Category

Open the Chinese Category Browser.

### Favourites

Show all favourite phrases.

### Settings

Allow the user to manage:

- Chinese categories
- Usage types
- Tone labels
- Export / import data

---

## 7. Responsive Design Requirements

The app must work well on both desktop and mobile.

### Desktop Web

Desktop layout should use two columns where possible:

- Left sidebar: categories
- Main area: phrase cards / editor

For the phrase editor:

- Left side: user input fields
- Right side: AI-generated examples and scenarios

### Mobile

Mobile layout should be single-column.

Requirements:

- Bottom navigation or simple top navigation.
- Large tap targets.
- Phrase cards should be compact.
- Category list should be easy to scroll.
- Copy buttons should be visible.
- Editor fields should not feel crowded.
- AI output should appear below the input fields.

---

## 8. UI Style Direction

The UI should feel clean, academic, calm, and personal.

Suggested style:

- Light background
- Soft blue / grey accent colour
- Rounded cards
- Clear typography
- Minimal visual noise
- Good spacing
- Mobile-first responsive layout

The interface should not look like a complicated database. It should feel like a personal writing notebook.

Suggested UI components:

- Phrase cards
- Category pills
- Tag chips
- Favourite star
- Copy button
- Expand/collapse sections
- Soft empty states

Example empty state:

```text
No phrases in this category yet.
Add your first phrase to build your personal writing bank.
```

## 11. Search and Filter

The MVP should support:

- Search by English phrase
- Search by English definition
- Search by tag
- Filter by Chinese category
- Filter by favourite

Advanced semantic search is not required for MVP.

---

## 12. Import / Export

MVP should include simple data export and import.

Export formats:

- JSON
- CSV

Import formats:

- JSON
- CSV

The export function is important because the phrasebank is personal long-term data and should not be locked inside the app.

---

## 13. Out of Scope for MVP

Do not include these in the first version:

- Automatic Chinese-to-English phrase search
- Full document rewriting
- Grammar checker
- Browser extension
- Word plugin
- Collaboration features
- User accounts
- Cloud sync
- Complex spaced repetition
- Citation management
- Large AI writing assistant

These can be added later.

---

## 14. Acceptance Criteria

The MVP is successful when the user can:

1. Add a new English phrase with an English definition.
2. Assign it to a Chinese category.
3. Ask AI to generate example sentences and usage scenarios.
4. Edit and save the AI-generated content.
5. Browse phrases by Chinese category.
6. Search phrases by keyword or tag.
7. Mark useful phrases as favourites.
8. Use the app comfortably on desktop web.
9. Use the app comfortably on mobile.
10. Export the phrasebank data as JSON or CSV.

---

## 15. Suggested Tech Stack

Use a simple web app stack.

Recommended:

- React or Next.js
- TypeScript
- Tailwind CSS
- Local storage or SQLite for MVP
- Optional: Supabase later if cloud sync is needed
- OpenAI API or another LLM API for example generation

For the MVP, local-first storage is acceptable. The most important thing is that the user can export their data.

## 17. Main User Flow

### Flow 1: Add a phrase

```text
User opens Add Phrase
→ enters English phrase
→ enters English definition
→ chooses Chinese category
→ optionally adds Chinese note and tags
→ clicks Save
```

### Flow 2: Generate examples

```text
User opens a saved phrase
→ clicks Generate examples with AI
→ AI returns example sentences, usage scenarios, misuse warning, and style note
→ user edits the output
→ user saves the phrase
```

### Flow 3: Browse while writing

```text
User opens Browse by Category
→ selects a Chinese category such as “限定 / 避免绝对化”
→ sees all saved English phrases under that category
→ expands a phrase card
→ reads examples and scenarios
→ copies the phrase or example sentence
→ uses it in their writing
```

---

## 18. Notes for Codex

Please implement the MVP as a clean, responsive, mobile-friendly web app.

Priority should be:

1. Data structure clarity.
2. Smooth phrase entry workflow.
3. Easy category browsing.
4. Good copy-to-clipboard experience.
5. Clean and calm UI.
6. Exportable user data.

Do not overbuild the first version. Keep the tool focused on personal phrase collection and browsing.
